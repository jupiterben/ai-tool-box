# 前端现代化优化快速开始指南

## 概述

本指南提供前端现代化优化功能的快速测试场景，用于验证功能是否按预期工作。

## 前置条件

1. 项目已安装依赖: `pnpm install`
2. 开发服务器可启动: `pnpm dev`
3. Electron 应用可正常运行

## 测试场景

### 场景 1: 主题切换功能

**目标**: 验证明暗主题切换功能正常工作

**步骤**:
1. 启动应用: `pnpm dev`
2. 打开应用界面
3. 查找主题切换按钮（通常在设置或侧边栏）
4. 点击切换按钮，从亮色主题切换到暗色主题
5. 观察界面变化

**预期结果**:
- ✅ 主题切换流畅，无闪烁
- ✅ 所有组件（侧边栏、主内容区、按钮等）颜色正确更新
- ✅ 文本清晰可读（对比度符合 WCAG 2.1 AA）
- ✅ 图标和视觉元素正确显示

**验证点**:
- [ ] 背景颜色正确切换
- [ ] 文本颜色正确切换
- [ ] 边框和分隔线颜色正确
- [ ] 按钮和交互元素颜色正确
- [ ] 图标颜色正确（使用 currentColor）

---

### 场景 2: 页面切换动画

**目标**: 验证页面切换时的过渡动画

**步骤**:
1. 启动应用
2. 在侧边栏选择不同的工具页面
3. 观察页面切换过程

**预期结果**:
- ✅ 页面切换有平滑的淡入淡出或滑动动画
- ✅ 动画时长适中（约 200-300ms）
- ✅ 动画流畅，帧率保持在 60fps
- ✅ 无卡顿或延迟

**验证点**:
- [ ] 动画流畅，无掉帧
- [ ] 动画时长合适，不会感觉太慢或太快
- [ ] 动画方向一致（如统一从右向左滑动）

---

### 场景 3: 交互元素反馈

**目标**: 验证按钮和交互元素的悬停、点击反馈

**步骤**:
1. 启动应用
2. 将鼠标悬停在侧边栏按钮上
3. 点击按钮
4. 观察反馈效果

**预期结果**:
- ✅ 悬停时有视觉反馈（颜色变化、轻微位移等）
- ✅ 点击时有明确的反馈（按下效果）
- ✅ 反馈延迟 < 100ms
- ✅ 反馈效果自然，不过度

**验证点**:
- [ ] 所有可交互元素都有悬停效果
- [ ] 点击反馈清晰可见
- [ ] 反馈响应迅速
- [ ] 反馈效果符合设计规范

---

### 场景 4: 响应式布局

**目标**: 验证不同窗口尺寸下的布局适配

**步骤**:
1. 启动应用
2. 调整窗口宽度到不同尺寸:
   - 800px（最小宽度）
   - 1024px（侧边栏折叠断点）
   - 1440px（标准宽度）
   - 2560px（大屏幕）
3. 观察布局变化

**预期结果**:
- ✅ 在 800px 宽度时，布局正常显示
- ✅ 在 1024px 以下时，侧边栏可折叠或隐藏
- ✅ 内容区域自适应可用空间
- ✅ 文本和控件在不同尺寸下保持可读性

**验证点**:
- [ ] 最小宽度（800px）下布局正常
- [ ] 侧边栏在 < 1024px 时可折叠
- [ ] 内容区域自适应
- [ ] 文本大小合适，不会过小或过大
- [ ] 控件大小合适，易于点击

---

### 场景 5: 键盘导航

**目标**: 验证所有功能支持键盘操作

**步骤**:
1. 启动应用
2. 使用 Tab 键在界面中导航
3. 使用 Enter 键激活按钮
4. 使用 Esc 键关闭对话框（如果有）
5. 使用方向键导航列表（如果有）

**预期结果**:
- ✅ Tab 键可以访问所有可交互元素
- ✅ 焦点指示器清晰可见（至少 2px 边框）
- ✅ Enter 键可以激活按钮和链接
- ✅ Esc 键可以关闭模态框
- ✅ 焦点顺序逻辑合理

**验证点**:
- [ ] 所有按钮可通过 Tab 访问
- [ ] 焦点状态清晰可见
- [ ] 键盘操作与鼠标操作功能一致
- [ ] 焦点顺序符合逻辑（从左到右，从上到下）

### 场景 6: 性能指标

**目标**: 验证性能指标符合要求

**步骤**:
1. 打开浏览器开发者工具
2. 切换到 Performance 标签
3. 刷新页面，记录性能指标
4. 使用 Lighthouse 进行性能审计

**预期结果**:
- ✅ FCP (First Contentful Paint) < 1.5s
- ✅ LCP (Largest Contentful Paint) < 2.5s
- ✅ FID (First Input Delay) < 100ms
- ✅ Lighthouse 性能评分 ≥ 90

**验证点**:
- [ ] FCP 时间符合要求
- [ ] LCP 时间符合要求
- [ ] FID 时间符合要求
- [ ] Lighthouse 评分达标

---

### 场景 7: 可访问性检查

**目标**: 验证可访问性符合 WCAG 2.1 AA 标准

**步骤**:
1. 安装 axe DevTools 浏览器扩展
2. 打开应用
3. 运行 axe 扫描
4. 检查颜色对比度

**预期结果**:
- ✅ 无严重可访问性问题
- ✅ 所有文本颜色对比度 ≥ 4.5:1（普通文本）或 ≥ 3:1（大文本）
- ✅ 所有交互元素有适当的 ARIA 标签
- ✅ 语义化 HTML 结构正确

**验证点**:
- [ ] axe 扫描无严重问题
- [ ] 颜色对比度达标
- [ ] ARIA 属性正确
- [ ] 语义化标签使用正确（nav, main, aside, button 等）

---

- ✅ 间距和布局一致
- ✅ 圆角和阴影效果一致
- ✅ 图标风格一致

**验证点**:
- [ ] 颜色方案统一
- [ ] 字体系统统一
- [ ] 间距系统统一
- [ ] 视觉元素风格统一

## 常见问题

### Q: 主题切换后某些组件颜色不正确？
**A**: 检查组件是否使用了 CSS 变量，确保所有颜色值都来自设计令牌。

### Q: 动画不流畅？
**A**: 检查是否使用了 `transform` 和 `opacity`，避免使用会触发重排的属性（如 `width`, `height`, `top`, `left`）。

### Q: 键盘导航无法访问某些元素？
**A**: 检查元素是否有 `tabindex` 属性，确保可交互元素使用语义化标签（如 `button` 而非 `div`）。

### Q: 响应式布局在小屏幕下显示异常？
**A**: 检查 CSS Media Queries 是否正确设置，确保断点值正确。
